AURA EVENTS - WEDDING & EVENT PLANNING PLATFORM
SYSTEM TECHNICAL SPECIFICATIONS
================================================================================

1. Project Introduction
Aura Events is a high-end, decoupled digital workspace and marketplace built specifically for couples, professional planners, and wedding vendors. The system is designed to streamline event organization, guest list management, real-time budget tracking, direct messaging, and secure online bookings and payments.

1.1 Product Purpose
Aura Events serves as a premium, end-to-end digital coordinator for weddings and formal events. The platform solves the coordination overhead by consolidating task schedules, vendor marketplace listings, budgeting sheets, and real-time RSVPs into a cohesive web application. It provides:
- Planners and couples with a modern, glassmorphic dashboard to track wedding milestones and manage guest databases.
- Wedding vendors with a dedicated shop portal to list services, receive booking requests, and request electronic payments.
- Platform administrators with governance tools to monitor events, manage users, and approve vendor services.

1.2 Product Scope
Aura Events comprises the following architectural layers:
- Aura Events React Frontend: A client-side Single Page Application (SPA) designed with an elegant, Apple-inspired editorial theme using Tailwind CSS v4, compiled and served via an Nginx container inside Kubernetes.
- Aura Events Laravel API Backend: A secure RESTful API built on Laravel 12 that processes business logic, handles database migrations, executes automated mail queues, and integrates with external payment gateways.
- Aura Events MySQL Container: A Dockerized MySQL 8.0 database container that stores and persists relational schemas for users, events, guests, budgets, service listings, bookings, messages, and transaction logs.
- Deployment Infrastructure: A single-node lightweight Kubernetes (K3s) cluster hosted on an AWS EC2 instance (8 vCPUs, 32 GB RAM, 40 GiB EBS volume). The cluster uses Nginx as the Ingress Controller to route external traffic, with the frontend React SPA served via Nginx container pods, and backend Laravel API and MySQL 8.0 database containers running as workloads within K3s, built automatically using a Jenkins CI/CD pipeline.

Aura Events Capabilities:
- Event and Schedule Management: Planner-side workspace for event CRUD, checklist tasks, and timeline programs.
- Guest Administration: Guest database management, bride/groom side allocation, and automated personalized SMTP invite dispatches.
- Public RSVP System: Unique guest token generation and public online form processing for confirmations, food preferences, and custom messages.
- Marketplace & Service Bookings: Category-filtered marketplace directory enabling location-aware searches, direct planner-to-vendor booking requests, and vendor accept/decline workflows.
- Financial Tracker & Razorpay Integration: Real-time budget monitoring and card/UPI payment processing with cryptographic signature verification and PDF invoice generation.
- Real-Time Direct Chat: In-app instant messaging between planners and vendors.
- Platform Auditing: Administrator dashboard for vendor verification and system-wide monitoring.

Product Boundaries (What the platform will not do):
- The platform will not support physical inventory management (e.g. tracking physical quantities of tables or chairs).
- The system will not process manual or off-platform cash transactions; all payments must be routed through the Razorpay SDK to maintain status tracking.
- The platform does not handle external travel or hotel room bookings.

1.5 Architectural Organization
The Aura Events system is organized into decoupled layers:
- Ingress Layer: Nginx Ingress Controller running in the K3s cluster that handles SSL/TLS termination and routes API traffic (`/api/*`) to the backend and UI traffic (`/*`) to the frontend.
- Presentation Layer: Client-side browser SPA running React and Tailwind CSS v4, served via Nginx container pods inside the K3s cluster, communicating with the server exclusively via JSON API requests.
- Application Logic Layer: Server-side Laravel API running inside dedicated K3s pods using Alpine PHP 8.2 containers.
- Persistent Data Layer: Relational MySQL 8.0 database running inside a K3s container with persistent volume storage mounted to the host's 40 GiB EBS volume.
- External Integration Layer: Secure webhooks and API calls routing through SSL/TLS connections to the Razorpay Payment Gateway and SMTP Mail relay servers.

2. General Description

2.1 Product Perspective
Aura Events is a decoupled, highly responsive client-server web application. The system is deployed on a single, high-performance AWS EC2 instance (8 vCPUs, 32 GB RAM, 40 GiB EBS volume) orchestrating containerized microservices using a lightweight Kubernetes (K3s) cluster. It uses Nginx as the Ingress Controller for single-point routing and secure SSL/TLS termination. Low-latency internal networking between the Nginx-served frontend SPA, Laravel API, and MySQL 8.0 containers is handled natively by K3s Service networks, backed up by persistent host-path mapping on the 40 GiB EBS volume.

2.2 Product Functions
The core capabilities of the Aura Events platform include:
- Security & Roles: Account creation with Sanctum token auth, distinguishing Planners, Vendors, and Admins.
- Workspaces: Management of events with durations, task checklists with due dates, and custom timeline schedules.
- Guests & RSVPs: Custom guest lists with bridal/groom side groupings, secure token-based invitation dispatches, and public mobile-responsive RSVP submissions.
- Marketplace: Vendor portfolios (descriptions, pricing, location, cover photos), service filters, and booking pipelines.
- Payments: Payment request generation, Razorpay checkout integrations, transaction verification via cryptographic signature checks, automatic budget calculations, and downloadable invoices.
- Communication: Direct messaging (chat) and real-time dashboard notifications.
- Admin Panel: Vendor application approval flows, user controls, and system metrics.

2.3 User Characteristics
- Planners & Couples: Demand editorial luxury visuals, rapid page loading (sub-second UI updates), and an elegant, stress-free user experience.
- Vendors: Require clear, mobile-friendly control panels to manage their service catalogs, review booking requests, send invoices, and track payment receipts.
- Administrators: Demand a secure, administrative portal with clear dashboard metrics, database access checks, and simple vendor approval buttons.

2.4 Technical and Operational Constraints
- Infrastructure: Deployment is limited to a single AWS EC2 virtual server (8 vCPUs, 32 GB RAM, 40 GiB EBS volume).
- Cluster Orchestration: All workloads must run as containerized services inside a single-node lightweight Kubernetes (K3s) cluster.
- Ingress Management: External HTTP/HTTPS routing, path partitioning, and SSL certification are strictly managed by an Nginx Ingress Controller.
- Database: Restricted to a MySQL 8.0 Docker container workload inside the K3s cluster, utilizing persistent directories mapped to the host's EC2 EBS storage.
- Framework: Core logic must remain locked to Laravel 12 (PHP 8.2+) and React 18+ to maintain security and codebase stability.
- Styles: Theme styles must remain restricted to Tailwind CSS 4 and Vanilla CSS variables to preserve the editorial design system.

2.5 Assumptions and Dependencies
- Host Server & Cluster: Assumes the AWS EC2 instance and the local K3s control plane remain operational. EBS volume performance must support adequate I/O speeds for database operations.
- Payment Services: Capture of online payments depends entirely on the availability of Razorpay services.
- Email Service: The SMTP server credentials in the backend environment file must be valid to permit automated guest invite dispatches.
